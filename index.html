「WEBコンテンツ作成」の成果

本テーマについて

学習目標:

HTMLによってWebページの構造を理解して記述する基本的な方法を理解し、CSSによってWebページを装飾して見た目を作る方法を理解する。 また、JavaScriptによってWebページの動きの処理をする記述の方法を理解し、簡単なWebページが作成できるようになる。

プロジェクト課題について

プロジェクト課題では「世界の天気」を選択した。 気候情報を知りたい都市を選択するとその都市の天気・最低気温・最高気温・湿度が表示される。 都市を選択し、「検索」ボタンを押すことで結果が得られる。 都市名ごとにvalueが設定されており、選択されることで検索キーに値が設定され、それを使って通信して情報を得ることができる。 表示される情報は、「都市名」「天気」「最低気温」「最高気温」「湿度」の５つである。 また、コンソール上にも気候情報と通信の結果が表示される。

プロジェクト課題で作成したページ: project.html

数当てゲーム

ランダムで生成される1~10の数を三回まで予想して答えと一致しているかを判定する数当てゲームを制作する。 第三回課題では予想の値を４に固定して「予想した回数」と「予想が正解しているか」によって変化する文章をコンソール上で表示されるようプログラムした。 また、不正解の場合は答えを上回っているか下回っているかを判定してヒントを出し、正解の場合は終了するようにした。 第四回課題ではDOMを使用して数当てゲームの結果をコンソール上ではなくページに表示されるよう修正した。予想は４で固定され、判定の回数を一回に減らした。 第五回課題では予想した数値を入力するテキストボックスと回答するボタンを設置し、予想を打ち込めるようにした。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

第1回講義では「事前準備の確認」、「Webページとは」、「HTMLについて」を学んだ。 事前準備の確認では、firefox、vscode、githubの設定について確認した。 WebページについてはURLの構成部品について学んだ。 HTMLについては、HTML要素、HTML文書の構成について学び、演習を行なった。

演習の成果：演習1-4: HTMLのさまざまな装飾

第2回 CSS による Web ページの装飾

第2回講義ではCSSによって見た目を装飾する方法について学んだ。 CSSをHTMLに適用する方法は外部スタイルシート、内部スタイルシート、インラインスタイルの三つがあるが、 基本的に外部スタイルシートを使用する。 CSSはセレクタ、プロパティ名、プロパティ値によって構成される。

演習の成果 : 練習問題2-3

第3回 JavaScriptの基本

第3回講義ではJavaScriptについて簡単な演習を通して学んだ。 JavaとJavaScriptは共通する点もあるが、異なる言語である。 HTMLにJavaScriptを導入するには三つの方法があるが、 演習では主に外部でファイルを作成する方法を使う。 Firefox上のウェブ開発ツールからコンソールを開くことでJavaScriptの実行を確認することができる。 一度そこで実行してからVScodeに書き込むことが望ましい。

演習の成果 : 練習問題3-6

第4回 DOM と JavaScript

第4回講義ではDOMについて学んだ。主に演習を中心に進め、プロジェクト課題、数当てゲームの改良を行なった。 DOMによって、要素の検索、テキストの変更、CSSプロパティの変更、二つの方法での新しい要素の追加、要素の削除を行うことができる。

演習の成果 : 練習問題4-2から4-5

第5回 イベント処理

第5回講義ではイベントが発生したときにプログラムを動作させる処理の方法について学んだ。 イベントとはボタンをクリックする、マウスポインタが重なる、キーを押すなどの操作のことである。 それによってテキストボックスやボタンを設置することができる。

演習の成果 : 練習問題5-3

工夫点，苦心点，強調点，感想

全体を通して、見やすいCSSを作れるよう工夫した。indexではできるだけシンプルに見出しごとに区切ったが、小見出しとなる学習内容の説明は他より目立って一目でわかるようなCSSを作れるよう意識した。プロジェクト課題では中央に大きなフォントサイズで表示されるようCSSを作った。 CSSを作り込むのは難しく、もっとやれたのではないかという点はあるが方法がわからず、客観的に見ることの難しさ、想定を実現することの難しさを実感した。 第六回講義でのプロジェクト課題について、ボタンが押されてから通信が行われて結果を得るところまでは順調であったが、2回以上検索を行うとそれ以前の結果が残り続けてしまう問題が発生し、解決に非常に苦労した。 原因は、結果を表示するプログラムをまとめ、通信成功時にprintを行う方法をとっていたが、その中で毎回要素の作成が行われていたことにあった。そのため、ひとまとめにしていた結果を表示するプログラム群からqureate文を外に出すことで解決した。 そのほかにも細々としたミスは多かったが、その多くがスペルミスであった。また、先にあげたあげた苦心については、ボタンが押された後のイベント処理に問題があると信じ込んでそれ以前の問題に気づけなかったのが大きなタイムロスとなった。 問題の解決においてメインの動作に疑いをかけるよりもさほど重要ではないところに問題が埋まっていることが多々あり、プログラムの動作においてはそのような小さなミスが最も厄介であると強く実感した。 全六回のWebコンテンツの作成を通して、Webサイトの仕組み、装飾、プログラム、通信などWebサイトの作成に必要な基礎的なことを学ぶことができた。 私にとってWeb作成は非常に興味がある分野だった。インターネットが普及して容易に様々なサイトにアクセスでき、幼い頃から利用してきたが、その作り方や見えないところでどんな動きをしているかなどは全く知らなかった。 そのため、閲覧者が目にするサイトの見栄えを作るCSSの仕組みを知れたことでこれから閲覧するサイトの見方が一つ増えたと思う。また、サイトの制作はどちらかといえばデザインの分野だと以前まで考えていた。 もちろん、サイトの見やすさや視覚効果など、デザインの技術も必要であると思うが、それ以上にサイトの根幹として情報技術が必要不可欠であると実感することができた。 情報の技術がいかに今の社会を支えているかを実感することにもつながった。
